# User testing notes

Informal usability sessions during Enclav development.

## Summary

- **Tested with:** 3 developers
- **Outcome:** All three successfully connected a wallet and completed repository scans end-to-end.

## Feedback and product changes

| Feedback | Change |
|----------|--------|
| Scan time feels long | Added an **elapsed timer** during scans so progress feels concrete (dashboard). |
| Unclear what **TEE** means | Added **in-UI explanation** of trusted execution / TeeML context so judges and users see why attestation matters. |
| Wanted to know **cost upfront** | Added **cost transparency** (credits / per-scan cost surfaced before and during use). |

These notes are for hackathon judges and future contributors: they tie observed friction to shipped UI and copy updates.
